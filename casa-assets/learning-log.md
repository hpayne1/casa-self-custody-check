## Notes

- Prefer **few, strong rules** over many weak ones.
- If feedback is "this run only," don't promote it to rules.

## 2026-02-21 — ExchangeHack — Head_of_marketing

### Contract
- Title: Exchange hack response (take-home preset)
- Deliverables: 1 single X post + 1 thread (≤5) + 1 blog (500–800) + 1 follow-up idea + what-we-wouldn't-do
- Key constraints (top 5):
  - no dunking / no schadenfreude
  - no panic CTAs ("move now")
  - no guarantees; security = risk reduction
  - write to impacted + spectators (avoid minimizing)
  - institutional calm + practical steps

### What worked
- "First-aid" single post + step-by-step thread reduced redundancy and increased usefulness.
- Visuals improved clarity when treated as frameworks (questions + checklist) rather than "news graphics."

### Rules for next time

**global_CASA_quality**
- Emojis are default **none**; exception: a single `🧵` is allowed at the start of an X thread opener as a structural cue. No emojis in on-image copy for crisis/security visuals.
- Visual briefs must include: purpose, **verbatim on-image copy**, layout notes, style notes, constraints, and export specs (designer handoff ready).

**scenario_specific** (only when `Scenario_tag=ExchangeHack`)
- If visuals are used, default to the **2-card "Self-custody check" pack** in `casa-assets/visual-assets.md` (Diagnostic questions + 4-step checklist). Use the on-image copy **verbatim**.
- Card 1 (questions) has **no CTA**; Card 2 is a calm framework (no urgency language).
